4.1 Collaborative Exercise

The purpose of this exercise is to put everything into practice that weâ€™ve learnt in exercises 1, 2 and 3, as well as learn about merging, merge conflicts and pull requests. This is key to collaborative projects in Git.

This part of the lesson is going to be less structured, and you'll be expected to know which commands to run for concepts we covered in previous lessons, but will still take you through the steps for new concepts.

Exercise Prerequisite

As this is a collaborative exercise, you will need to organise yourselves into small groups of 2-3, and choose one person in your group to be a team leader.

Lesson Objectives

  • Fork and clone the collaborative exercise GitHub repository
  • Learn how to invite people to a repository to work collaboratively
Step 1 of 10

Fork the Repository

We've already looked at how to fork a repository, so you should be an expert at this by now.

The only difference when forking a repoistory in a collaborative project is that only one person needs to fork it, and this person should be your team leader.

This person will also be the owner of the repository.

  • Fork this repository for exercise 4
Step 2 of 10

Clone The Forked Repository

File path you need to be in: "C:\Users\YourName\git-exercises"

Run the following commands in Git Bash

git clone forked_repo_link

This will create a folder named git-exercise-04

Step 3 of 10

Create a New Empty GitHub Repository

Your designated team leader who also forked the repository should be the one to create the new GitHub repository.

  • Copy the forked repo link
  • Go to GitHub and click 'New repository'
  • Give your repository a name e.g. git-team-exercise
  • Set the repository to public
  • Do not initialise the repository with a README as we will be adding files later
  • Click 'Create repository'
Step 4 of 10

Create a New Folder On Your Local Machine

To move the files to the empty GitHub repository, you need to create a corresponding folder on your local machine for the new GitHub repository. Then we need to initialise it.

File path you need to be in: "C:\Users\YourName\git-exercises"

Run the following commands in Git Bash

mkdir git-team-exercise
cd git-team-exercise
git init
Step 5 of 10

Move Files from Forked Repository to the New Repository

Because we want the original repository to remain untouched when we come to pushing changes and merging branches into master, we need to copy the files which provide the template for the collaborative exercise into the new repository.

  • Open your local directory where the git-exercise-04 folder (cloned fork) is located
  • Copy all the files from git-exercise-04 folder (Ctrl + A, Ctrl C) to the new repository folder called git-team-exercise
  • Paste the files (Ctrl + V) into the git-team-exercise folder
Step 6 of 10

Add the Remote Repository for the New Repository

After moving the files, go into the git-team-exercise folder in Git Bash so we can add the new remote repository we created on GitHub.

File path you need to be in: "C:\Users\YourName\git-exercises\git-team-exercise"

Run the following commands in Git Bash

git remote add origin new_github_repo_link
Step 7 of 10

Create an Initial Commit

As our repository has no commits yet, we need to create an initial commit before we can push to the remote repository.

File path you need to be in: "C:\Users\YourName\git-exercises\git-team-exercise"

Run the following commands in Git Bash

git add .
git commit -m"Initial commit"
Step 8 of 10

Push The Files to the New GitHub Repository

Now that we've linked our local repository where we copied and pasted files from the forked repository, we can now push the files to the new GitHub repository.

File path you need to be in: "C:\Users\YourName\git-exercises\git-team-exercise"

Run the following commands in Git Bash

git push -u origin master

What is this code doing?

  • git push -u origin master: This pushes the cloned forked repository back to GitHub. -u links the local master branch to the remote master branch.
Step 9 of 10

Invite Collaborators

As this is a collaborative task, we will need to invite other people.

  • Click on 'Settings' in your git-team-exercise GitHub repository
  • On the left, click on 'Collaborators'
  • Click on 'Add people'' and invite your colleagues to the repository
Step 10 of 10

Accept the Invitation

If your team leader has invited you to collaborate on the GitHub repository, you will now need to accept the invite.

Either login to your email or go to the notification tab on your GitHub account to view the invite link.

  • Click on the invite link
  • Select 'Accept invitation'

Now you should be able to gain access to the repository and also be able to push to the repository.

Lesson Summary

In this lesson, we explored the process of collaborating on a Git project by forking and cloning repositories. The first step involved forking the main repository, after which the team leader cloned the repository to their local machine. We also learned how to create a new GitHub repository, add a new folder on the local machine, and move the necessary files from the forked repository to the new repository.
