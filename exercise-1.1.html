1.1 Creating and Initialising a Local Repository

In this lesson, we will learn how to create and initialize a local Git repository. This is the first step in using Git for version control, allowing us to track changes in our projects. We will cover how to create a new directory, navigate into it, and set up Git tracking using git init.

Lesson Objectives

  • Create a new local project directory
  • Navigate into a directory
  • Initialise a Git repository
  • Understand why initialising a repoistory is important for tracking changes
Step 1 of 4

Creating a Local Repository

A Git repository is a storage location for a project and it's tracked using Git. It allows multiple people to collaborate on the same file, track modifications and revert to previous versions

First, we'll start off by creating our own directory

Start by opening Git Bash in a suitable location on your computer - ideally somewhere outside of OneDrive, such as your User Directory, usually found at C:\Users\YourName

Avoid using OneDrive because it automatically syncs files, which can interfere with Git's version control and lead to issues

Run the following commands in Git Bash

mkdir git-exercises
cd git-exercises
mkdir git-exercise-01
cd git-exercises

This will create a folder called 'git-exercises' and this is where we'll store all of our work. We've created a nested folder to store all the files for exercise 1 and navigated into the directory.

Your command line should say you're in the 'git-exercise-01' directory.

Step 2 of 4

Initialise the Repository

Initialising a repository creates a new Git repository in a folder, and sets up Git tracking for a project. It creates a hidden .git folder that stores version history. After initialising your repository, you can starting adding, committing, and managing versions of your files.

Run the following command in Git Bash

git init

The terminal should display that you've initialised an empty repository and set up the .git directory. .git is a hidden folder, which contains all the files for version control, meaning that your project can now be tracked by Git.

Step 3 of 4

Verify the .git Folder

You should now have a hidden .git folder that stores version history.

We can verify this by looking inside the folder.

Using File Explorer:

  • Open File Explorer
  • Navigate to the directory where you initialised the Git repository (git-course\git-exercises\git-exercise-01)
  • Click on the View tab at the top of the File Explorer window
  • Check the box labeled Hidden items to show hidden files and folders
  • You should now see the .git folder in the directory

Lesson Summary

In this lesson, we learned how to create a local Git repository from scratch. We started by using the mkdir command to create a new directory and cd to navigate into it. Then, we initialised an empty Git repository using git init, which set up the necessary .git folder to track changes. This process allowed us to turn any directory into a version-controlled project.
