4.2 Making Changes on Branches

The purpose of this exercise is to put everything into practice that we’ve learnt in exercises 1, 2 and 3, as well as learn about merging, merge conflicts and pull requests. This is key to collaborative projects in Git.

Lesson Objectives

  • Learn how to create your own branch to make changes on
  • Edit the 'Meet The Team' page and add a section about yourself
Step 1 of 7

Open the Repository files in a Code Editor

Using File Explorer:

  • Search for the git-exercises folder
  • Click on the git-team-exercise folder
  • Right click on the index.html file
  • Open the file with your preferred code editor
Step 2 of 7

Create Your Own Branch

If everyone edits the same file directly in the master branch, Git might struggle to later merge changes properly, leading to merge conflicts.

By working in seperate branches, each person can edit their section without interfering with others.

Now we're all going to create our own seperate branches.

Run the following commands in Git Bash

git checkout -b YourName-feature

Insert your own name

Step 3 of 7

Open the Code in a Web Browser

To visually track changes as we go along, we need to have the code open in a web browser.

  • Search for the git-exercises folder
  • Click on the git-team-exercise folder
  • Right click on the index.html file
  • Open the file with your preferred web browser
  • Click on the 'Meet The Team' page in the top navigation bar on the website
Step 4 of 7

Editing Lines of Code

Everyone is going to have their own section of code to edit, so make sure you're on the correct section for yourself.

Let's see an example of how each person will edit their 'about me' section.

  • Click on team.html in the explorer on the left of VS Code
  • Find the first section of code which is documented as 'About me 1'
  • Replace anywhere you see ‘Forename Surname’ and ‘DATA ANALYST’ in this section of code, with your name and job title.
  • On line 30, replace the existing description encapsulated in single quotes, with a short description of yourself
  • You can also insert an image of yourself if you have one
  • Save the file (Ctrl + S)
  • Refresh the browser and ensure that your section has been updated
Step 5 of 7

Stage, Commit & Push Your Changes

Now that you've edited your own section, we need to stage and commit this change to the branch your created.

Run the following commands in Git Bash

git add .
git commit -m"Added information to meet the team section"
git push origin your_branch_name
Step 6 of 7

Refresh the webpage

To double check that we succesfully saved our changes, refresh your web browser.

Note: You'll only see your changes because you're in your own branch of development.

Step 6 of 7

Merging the Feature Branch into the Master Branch

If you're happy with how your section looks, we go ahead and merge this into the master branch.

  • Open your git-team-exercise repository in GitHub
  • Click on the button at the top of the repository called 'Compare & pull request'
  • Scroll down and click 'Create pull request'
  • GitHub should state that there is no conflict with the base branch so now click 'Merge pull request'
  • Change the commit message if you want to, then click 'Confirm merge'
  • Your pull request should have successfully merged
  • Now you can click 'Delete branch' as we've implemented our new feature
Step 7 of 7

Pull the Latest Changes

Everyone in your team should have pushed their sections to the remote GitHub repository however, you can still only see your changes.

This is because we don't have the most up to date version of the files in our local repository, which doesn't get updated automatically. We have to manually retrieve the latest changes from the remote repository.

Run the following commands in Git Bash

git pull origin master

Now when you refresh the webpage, you should see the changes made by everyone else in your team.

Your code should be updated too.

Lesson Summary

  • Learned how to create a new branch for making changes.
  • Created and switched to a personal branch.
  • Edited the "Meet The Team" page with personal details.
  • Merged the feature branch into the master branch.
  • Pulled the latest changes to update the local repository.
