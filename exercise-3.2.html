Why wouldn't we initialise the repository with a README?

Whether or not to initialise a repository with a README file depends on the use case. Here's some reason why you might choose not to initialise a repository.

When to Initialise with README When NOT to Initialise with README
You are starting a new project from scratch You are pushing an existing project to the repo
You want the repo to be easily browsable on GitHub You are setting up a bare repository
The project is open-source and needs documentation from the start Your team prefers to add the README in the first meaningful commit
You want contributors to understand the project immediately The project structure requires manual file organization first

In our case, we want to push an existing project to the repository, so we won't intialise it with a README.
