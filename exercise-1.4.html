1.4 Modifying Commits

So far, we've made three commits to our local repository. The first was our initial commit, in the second commit we answered questions 1-3 on the .txt file and in the third commit, we answered questions 4 and 5.

What if we wanted to edit one of those previous commits?

Being able to edit a commit in Git is useful for maintaining a clear and accurate project history. It helps ensure that your commits are well-structured, meaningful and free from error.

To edit a specific commit in Git, you have a few options depending on whether it's the most recent commit or an earlier commit.

Lesson Objectives

  • Understand the importance of editing commits
  • Use git commit --amend to modify the most recent commit
  • Use git rebase to edit an older commit
Step 1 of 4

Modifying the most recent commit

Let's imagine we made a mistake in the message of our most recent commit. We can use the git --amend command modify this.

Similar to the commit command, we can use the flag -m to specify a new commit message.

Run the following command in Git Bash

git commit --amend -m"Answered questions 4 and 5 on Trivia file"

By default, the ammend flag will edit the most recent commit.

Step 2 of 4

Verify The New Commit Message

Now that we've changed the commit message of our most recent commit, how can we verify that it was modified?

This is where the git log command comes in.

git log shows the commit history of a repository, allowing you to view past commits along with details like commit messages, authors, timestamps, and commit SHAs.

Run the following command in Git Bash

git log

As you can see, git log shows a list of commits in chronological order including the SHA, author, date of the commit and the message.

git commit --amend

This is the most basic git log command. We can use additional flags with this command to retrieve specific information about a commit. We'll be taking a further look at these in later lessons.

Step 3 of 4

Modifying Files in a Previous Commit

Let's take a look at how to edit contents of a file from a previous commit that wasn't the most recent one.

After runing git log, we can see that every commit has a SHA which uniquely identifies that commit. We can use this SHA to target a specific commit we want to modify.

We'll edit the commit where we answered questions 1-3.

  • Open the exercise_1_questions.txt file in Notepad
  • Edit the file. For simplciity, we'll just put the word 'The' in front of the first answer.
  • Save (Ctrl + S) and close Notepad.
  • Now we need to retrieve the SHA of the commit where we answered the first three questions

Run the following command in Git Bash

git log
git rebase -i HEAD~2
  • Copy the whole SHA of the specified commit.

Note: Your SHA will be different from mine because they're all unique.

Step 4 of 5

Commit The Change

A good commit message is essential for maintaining clarity and understanding in your project, especially when working in teams or revisiting code in the future.

What makes a good commit message?

  • Clear and Descriptive: Summarises what the change is
  • Short Message: Keep it brief (60 characters or less), but provide enough context
  • Focus on a Single Change: Each commit should represent one logical change. If it doesn't, then you should probably make more than one commit

Now that we've taken a look at some best practices for commit messages, let's create one ourselves.

Run the following command in Git Bash

git commit -m"Added answers to questions 1-3 on exercise_1_questions.txt"
Step 5 of 5

Answer the Rest of The Questions and Commit the Changes

Now that we've made some changes to the file and committed these changes, go ahead and answers questions 4 and 5 on the file, and then stage and commit the file as usual.

Tip: Don't forget to save (Ctrl + S) and close the file before staging it.

Run the following commands in Git Bash

notepad exercise_1_questions.txt
git add .
git commit -m"Added answers to questions 4 and 5 on exercise_1_questions.txt"

Lesson Summary

In this lesson, we learned how to modify Git commits to ensure a clean and accurate project history. We started by understanding the importance of editing commits to keep them meaningful and error-free. Using the git commit --amend command, we modified the most recent commit's message, and verified the changes with git log. We also explored how to edit earlier commits using git rebase -i, targeting specific commits with their unique SHA.
